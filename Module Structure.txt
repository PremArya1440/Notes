## 🔹 Root Module (dev/main.tf)

```hcl
module "vpc" {
  source = "../../modules/vpc"

  project         = "demo"
  environment     = "dev"
  cidr_block      = "10.0.0.0/16"
  public_subnets  = ["10.0.1.0/24"]
  private_subnets = ["10.0.2.0/24"]
}

module "ec2" {
  source = "../../modules/ec2"

  subnet_id  = module.vpc.public_subnets[0]
  project    = "demo"
  environment = "dev"
}

module "alb" {
  source = "../../modules/alb"

  public_subnets = module.vpc.public_subnets
}
```

# 🔥 2. Mock Interview (Real Questions + Answers)

## 🎯 Q1: What happens when you run `terraform init`?

👉 Answer:

> It initializes the project, downloads providers, and fetches modules from sources like GitHub.

---

## 🎯 Q2: Difference between module and resource?

👉 Answer:

> Resource creates infrastructure, while a module is a reusable collection of resources.

---

## 🎯 Q3: How do you pass data between modules?

👉 Answer:

> Using outputs from one module and passing them as inputs to another module.

Example:

```hcl
subnet_id = module.vpc.public_subnets[0]
```

---

## 🎯 Q4: What is state file?

👉 Answer:

> Terraform state file stores the current infrastructure mapping to track changes.

---

## 🎯 Q5: How do you handle multiple environments?

👉 Answer:

> Using separate folders (dev/prod) or workspaces.

---

## 🎯 Q6: What is remote backend?

👉 Answer:

> It stores state in remote storage like S3 for collaboration and locking.

---

## 🎯 Q7: What is `depends_on`?

👉 Answer:

> Explicit dependency between resources.

---

## 🎯 Q8: Difference between `count` and `for_each`?

👉 Answer:

> count is index-based, for_each works with maps and is more flexible.

# 🚀 3. Production-Level Terraform Best Practices

## ✅ 1. Use Remote Backend

```hcl
terraform {
  backend "s3" {
    bucket = "my-terraform-state"
    key    = "dev/terraform.tfstate"
    region = "us-east-1"
  }
}
```

---

## ✅ 2. Use `.tfvars` file

```hcl
project = "demo"
environment = "dev"
```

Run:

```bash
terraform apply -var-file="dev.tfvars"
```

---

## ✅ 3. Use Naming Convention

```hcl
Name = "${var.project}-${var.environment}"
```

---

## ✅ 4. Avoid Hardcoding

❌ Wrong:

```hcl
instance_type = "t2.micro"
```

✅ Better:

```hcl
instance_type = var.instance_type
```

---

## ✅ 5. Use Version Control (Git)

✔️ Store modules in GitHub
✔️ Use versioning (`?ref=v1.0.0`)

---

## ✅ 6. State Locking (Important)

👉 Use DynamoDB with S3
