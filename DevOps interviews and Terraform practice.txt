# 1️⃣ Terraform Architecture

Terraform is an **Infrastructure as Code (IaC) tool**.

Instead of creating servers manually in AWS Console, you write code.

Example:

```
AWS Console → Click → Create EC2
```

Terraform way:

```hcl
resource "aws_instance" "example" {
  ami           = "ami-0220d79f3f480ecf5"
  instance_type = "t3.micro"
}
```

Terraform will automatically create the server.

---

# 2️⃣ Terraform Project Structure

A real Terraform project looks like this:

```
terraform-project
│
├── provider.tf
├── variables.tf
├── terraform.tfvars
├── main.tf
├── outputs.tf
└── locals.tf
```

Explanation:

| File             | Purpose                      |
| ---------------- | ---------------------------- |
| provider.tf      | Cloud provider configuration |
| main.tf          | Main infrastructure code     |
| variables.tf     | Define variables             |
| terraform.tfvars | Assign variable values       |
| outputs.tf       | Display outputs              |
| locals.tf        | Define local values          |

# 3️⃣ Provider (Connect Terraform to AWS)

Example:

```hcl
provider "aws" {
  region = "us-east-1"
}
```

Explanation:

| Parameter | Meaning        |
| --------- | -------------- |
| aws       | Cloud provider |
| region    | AWS region     |

Terraform uses **AWS provider plugin**.

Before using provider we run:

```
terraform init
```

This downloads the AWS plugin.

---

# 4️⃣ Resource (Create Infrastructure)

Resources create real infrastructure.

Example EC2:

```hcl
resource "aws_instance" "example" {

  ami           = "ami-0220d79f3f480ecf5"
  instance_type = "t3.micro"

}
```

Explanation:

| Part         | Meaning           |
| ------------ | ----------------- |
| resource     | Terraform keyword |
| aws_instance | Resource type     |
| example      | Resource name     |

Full format:

```
resource "RESOURCE_TYPE" "NAME"
```

---

# 5️⃣ Variables (Reusable Infrastructure)

Variables allow dynamic configuration.

Example:

```
variables.tf
```

```hcl
variable "instance_type" {
  default = "t3.micro"
}
```

Use variable:

```hcl
instance_type = var.instance_type
```

Terraform automatically reads:

```
var.instance_type
```

---

# 6️⃣ terraform.tfvars (Provide values)

Example:

```hcl
instance_type = "t3.small"
```

This overrides default value.

Good for:

* dev
* staging
* production

Example:

```
dev.tfvars
prod.tfvars
```

# 7️⃣ Locals (Your Current Topic)

You are currently learning **locals**.

Example:

```
locals.tf
```

```hcl
locals {
  instance_name = "${var.name}-${var.environment}"
}
```

Example variables:

```
name = "locals"
environment = "dev"
```

Result:

```
locals-dev
```

Use locals like this:

```hcl
tags = {
  Name = local.instance_name
}
```

Difference:

| Variables       | Locals               |
| --------------- | -------------------- |
| External input  | Internal calculation |
| Defined by user | Defined inside code  |

# 8️⃣ Outputs (Display Information)

After infrastructure creation we display outputs.

Example:

```
outputs.tf
```

```hcl
output "public_ip" {
  value = aws_instance.example.public_ip

```
public_ip = 54.210.10.33

* CI/CD
* automation

# 9️⃣ Modules (Very Important)

Modules are **reusable Terraform code**.

Example:

Instead of writing EC2 code multiple times:

```
module "ec2" {
  source = "./modules/ec2"

  instance_type = "t3.micro"
}
```

Modules allow:

✔ Code reuse
✔ Standardization
✔ Large infrastructure management

Used heavily in **DevOps teams**.

# 🔟 Remote State (Real DevOps Setup)

By default Terraform stores state here:

```
terraform.tfstate
```

Problem:

Team members cannot share it.

Solution: **Remote State**

Example:

```
S3 + DynamoDB
```

Example configuration:

```hcl
terraform {
  backend "s3" {
    bucket = "terraform-state-prem"
    key    = "dev/terraform.tfstate"
    region = "us-east-1"
    dynamodb_table = "terraform-lock"
  }
}
```

Benefits:

| Feature            | Purpose            |
| ------------------ | ------------------ |
| S3                 | Store state file   |
| DynamoDB           | Lock state         |
| Team collaboration | Multiple engineers |

# 🧠 Important Interview Questions

### 1️⃣ What is Terraform state file?

Answer:

> Terraform state file stores the mapping between infrastructure and Terraform configuration.

Example:

```
terraform.tfstate
```

---

### 2️⃣ What is Terraform provider?

Answer:

> Provider is a plugin that allows Terraform to interact with APIs of cloud providers like AWS, Azure, or GCP.

---

### 3️⃣ What is difference between variable and local?

| Variable    | Local          |
| ----------- | -------------- |
| Input value | Computed value |
| External    | Internal       |
